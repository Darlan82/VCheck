# Contexto

Possuímos uma empresa de transporte que precisa fazer checklist de veículos diários após a saída e entrada de carregamento e descarregamento de produtos, onde o executor irá analisar e validar os dados do checklist se estão em conformidade ou não:

- **Itens a serem inspecionados** (ex.: faróis, substituição de itens)
- **Observação sobre os itens** (detalhes adicionais)

Para efetivar o checklist é necessário que o supervisor aprove ou reprove o checklist que foi executado.

---

# Implementação

Criar uma API Rest de execução completa de checklist.

---

# Validação

Garantir que a execução do checklist seja feita em partes, porém após iniciada por um executor, barrar a execução de um novo executor.

> Lembre-se de utilizar as melhores práticas de desenvolvimento de software.
